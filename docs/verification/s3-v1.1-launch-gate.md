# Sprint 3 v1.1 Launch Gate

Date: 2026-06-19
Updated: 2026-06-20

## Scope

This gate closes the deterministic local and fixture verification for Sprint 3. It verifies notification payload minimization, model interop fidelity, ChatGPT GA metadata packaging, local write-safety posture, and the expanded privacy scan.

It does not claim a live GPT App Directory listing, production Slack/email delivery, or production packet capture. Those remain external deployment artifacts and must be recorded separately before a production GA claim.

## Evidence

- `NotificationEvent` schema is a strict whitelist and rejects code, diff, finding, architecture body, and model body fields.
- Notification providers cover GitHub Check default, Slack, generic webhook with HMAC, and email with unsubscribe requirement.
- Notification provider configuration is opt-in and can be scoped to user or installation.
- Notification retry, dead-letter, and idempotency behavior is tested.
- Webhook HMAC signing requires a resolved secret value; `secret://` references are not used directly as signing keys.
- Control-plane notification queue stores the serialized minimal payload, not the caller-provided event object.
- LikeC4, Structurizr, and Mermaid export paths are deterministic and stable-sorted.
- LikeC4 and Structurizr import paths are initialization-only and strip Native protected fields.
- Native source-of-truth protection lives in `packages/architecture-domain`.
- `archctx export likec4|structurizr|mermaid` and `archctx import likec4|structurizr` are covered by CLI tests.
- ChatGPT Cloud Metadata App manifest, Directory listing metadata, review checklist, release rollback, and GA UI states are covered by tests.
- Secure MCP Tunnel remains loopback, scoped, short-lived, and revocable; write tools are disabled by default unless explicitly enabled.
- `bun run verify` now includes `tsc --noEmit`, so broken type-only imports are covered by the repo-local gate.
- `privacy-route-audit` now scans notification, adapter, renderer, CLI, control-plane, cloud metadata, GitHub App, and cloud DB surfaces.
- `docs/security/captures/sprint3-integrations.har.json` covers notification, Directory metadata, and rollback fixture traffic with metadata-only payloads.
