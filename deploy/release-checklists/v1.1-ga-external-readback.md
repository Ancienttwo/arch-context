# v1.1 GA External Readback Checklist

## Authority

External production GA requires evidence outside the local deterministic Sprint 3 suite. This checklist follows the current OpenAI Apps SDK submission surface:

- App submission and maintenance: https://developers.openai.com/apps-sdk/deploy/submission
- App submission guidelines: https://developers.openai.com/apps-sdk/app-submission-guidelines
- Apps SDK authentication: https://developers.openai.com/apps-sdk/build/auth

## Checklist

- [ ] `deploy/scripts/production-ga-readback.mjs preflight --environment production` returns `ready`.
- [ ] `https://archcontext.dev/privacy` or configured privacy URL returns a production privacy policy.
- [ ] `https://archcontext.dev/chatgpt/directory` or configured Directory metadata endpoint returns `slug=archcontext` and `repositoryContent=local-runtime-only`.
- [ ] OAuth discovery is readable and matches OAuth 2.1 / PKCE submission posture.
- [ ] GPT App Directory listing/readback evidence is recorded through `ARCHCONTEXT_OPENAI_DIRECTORY_URL` or `ARCHCONTEXT_OPENAI_DIRECTORY_EVIDENCE_PATH`.
- [ ] Real provider delivery evidence exists for at least one configured provider path without code, diff, finding, or architecture body payload.
- [ ] `docs/security/captures/production-redacted.har.json` or staging equivalent passes `node scripts/privacy-packet-capture-audit.mjs`.
- [ ] Capture manifest pending entry is converted to verified with `node scripts/privacy-capture-manifest.mjs record --environment production --capture docs/security/captures/production-redacted.har.json --id production.real-capture`.
- [ ] `node scripts/privacy-capture-manifest.mjs readback --require-environment production` reports production verified.
- [ ] `node scripts/security-scan-manifest.mjs readback --require-environment production` reports zero Critical and High production findings.
